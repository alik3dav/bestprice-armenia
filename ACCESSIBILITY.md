# Accessibility Standards

Aim for WCAG 2.2 AA. Use semantic HTML before ARIA. Buttons perform actions; links navigate. Forms require labels, accessible descriptions/errors, and logical grouping. Product grids, filters, tables, dialogs, drawers, menus, and carousels must be keyboard operable.

Focus must be visible and managed for dialogs/drawers. Do not trap focus except in modal contexts; restore focus when closing overlays. ARIA must enhance native semantics, not replace them unnecessarily. Text and interactive controls need sufficient contrast. Do not communicate price changes, discounts, availability, or validation solely by color.

Images need meaningful alt text or empty alt text when decorative. Dynamic updates should be announced when they change user task state. Touch targets should be comfortable on mobile, especially primary shopping actions.
