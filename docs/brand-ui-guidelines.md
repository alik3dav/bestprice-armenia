# Brand UI Guidelines

This compatibility guide supports `DESIGN-SYSTEM.md`, `UX-STANDARDS.md`, `COMPONENT-STANDARDS.md`, and `ACCESSIBILITY.md`. If rules overlap, follow the stricter standard.

## Marketplace UI direction

BestPrice Armenia UI must be compact, trustworthy, and comparison-first. Do not use one-off decorative styles that reduce product density or scanning speed.

## Required public patterns

- Header search remains prominent.
- Product cards include stable image area, title, rating/reviews when available, price, store count or availability, and optional discount/favorite/compare actions.
- Product grids default to 5 desktop, 3 tablet, and 2 mobile columns.
- Category/search pages include breadcrumbs, title, product count, sort, selected filter chips, desktop filter sidebar, and mobile filter drawer/bottom bar.
- Product detail pages prioritize title, media, best price, availability, merchant CTA, comparison table, specs, reviews, and related products.
- Store comparison rows emphasize store, price, delivery/availability, trust, and CTA.

## Token and styling rules

- Use documented color, spacing, radius, shadow, typography, and component variants only.
- Cards use white surfaces, subtle borders, compact padding, and restrained shadows.
- Buttons use shared primary, secondary, link, and destructive variants.
- Badges are semantic and compact; do not create decorative badge colors.
- Rating stars use the rating token and include numeric text when available.

## Localization and accessibility

Design for Armenian first, with Russian and English compatibility. Do not encode localizable text inside images. All controls require accessible names, visible focus, keyboard support, and sufficient contrast.

## UI review checklist

Verify documented tokens, grid behavior, mobile filters, reusable components, semantic HTML, focus states, Armenian text fit, no horizontal overflow, no heavy shadows, and no hidden applied filter state.
