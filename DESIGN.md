# Design Compatibility Guide

The authoritative design system is `DESIGN-SYSTEM.md`; UX rules live in `UX-STANDARDS.md`; reusable component rules live in `COMPONENT-STANDARDS.md`; accessibility rules live in `ACCESSIBILITY.md`. This file replaces the earlier Wise-inspired draft to remove conflict with the active marketplace brand guidance.

## Active direction

BestPrice Armenia uses a compact marketplace design language, not a generic fintech landing-page system. Public pages must prioritize search, categories, product grids, offer comparison, prices, merchant trust, SEO content, and mobile shopping utility.

## Core rules

- Use documented tokens only; do not introduce one-off colors, spacing, shadows, or card styles.
- Product grids default to 5 desktop, 3 tablet, and 2 mobile columns.
- Product cards stay dense: image, title, rating, price, availability/store count, and optional compact actions.
- Category/search pages use filter sidebar on desktop and drawer/bottom controls on mobile.
- Product detail pages surface title, media, best price, availability, and merchant CTA above the fold.
- Mobile must behave like a shopping app with prominent search and thumb-friendly actions.

For complete standards, read `DESIGN-SYSTEM.md`, `UX-STANDARDS.md`, and `docs/brand-ui-guidelines.md`.
